# 初识 CTF

## 什么是 CTF?

CTF（Capture The Flag）中文一般译作夺旗赛，在网络安全领域中指的是网络安全技术人员之间进行技术竞技的一种比赛形式。

参赛选手通过完成攻击、分析，解密等各类操作，率先从题目给出的沙箱环境或文件样本中得到一串具有一定格式的字符串或其他内容，并将其提交给平台，从而夺得分数。为了方便称呼，我们把这样的内容称之为"Flag"。Flag 的一般格式是一个统一的开头名称加上大括号内容，例如 `flag{hello_world}` 或者 `xxxctf{we1c0me!}`。

**简而言之**，flag 是一个答案，这个答案可能藏在任何一个规定的位置，需要你通过题目的关卡一步步拿到这个答案。

在 Web 方向中，CTF 题本质是 “模拟真实 Web 漏洞攻防”，聚焦于网站与 Web 服务的安全问题，题目会搭建一个存在漏洞的网站 / 应用，要求你**利用漏洞**获取 flag 。

## 练习平台

1. **[NSSCTF](https://www.nssctf.cn/)**  提供题库、比赛和 Writeup 资源，国内很多比赛的题目可以在这里复现，非常适合系统练习和社区交流。
2. **[BUUCTF](https://buuoj.cn/)** 包含大量 CTF 比赛题目的聚合平台，适合题海战术式练习和复现经典题目。
3. **[Hello CTF](https://hello-ctf.com/)** Hello CTF 是免费开源的 CTF 入门教程，拥有全面的知识点锦集、配套的题目靶场和完善的工具系统。
4. **[青少年 CTF 练习平台](https://www.qsnctf.com)** 提供题库、比赛和 Writeup 资源
5. **[CTFSHOW](https://ctf.show/)** 一个适合初学者的 CTF 练习平台

这些练习平台大多都需要注册账号和手机号认证，后续我们给予的练习也将来自这些平台，

因此可以提前进行相关账号的注册。

## 关于学习笔记

CTF 与 ACM 等算法类竞赛不同，它是**允许开卷**，且一般**允许断网在自己的电脑环境下解题**的比赛。

因此对于完成学习笔记，记录学习过程，建议大家不仅仅以完成本周作业为首要目的，更多的是

## 关于题目和实验

网络安全是**一门以实验为基础**的学科，作为网络安全的竞赛，我们平时的学习不能仅仅停留在理论，而是需要不断通过实践积累遇到的困难和问题的解决方式。

希望大家在书写 WriteUp 时尽量描述详细一些，既有自己的分析，又有看了他人 WP 后成功解题获得的启示，这些必然会让你加深对各个知识点的理解，提高做题熟练度。

也希望大家在学习完一个知识点后，能够及时动手完成布置的题目，并学会主动在各平台搜索含该知识点的新题，对其予以实践。
