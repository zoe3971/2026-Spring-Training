# 三、Linux基础指令
## 文件与目录操作
ls：列出目录内容

ls -l：显示详细信息（权限、大小、修改时间等）

ls -a：显示隐藏文件（以.开头）

cd：切换目录

cd ..：返回上级目录

cd ~：进入用户主目录

pwd：显示当前工作目录的绝对路径

mkdir：创建新目录

mkdir -p dir1/dir2：递归创建多级目录

rm：删除文件/目录

rm -r dir：递归删除目录及内容

rm -f file：强制删除（不提示）

cp：复制文件/目录

cp -r source_dir target_dir：递归复制目录

mv：移动/重命名文件

mv oldname newname：重命名文件

##    文件内容查看
cat：查看文件全部内容

more/less：分页查看文件（less支持上下滚动）

head/tail：查看文件开头/结尾部分



##    系统信息与管理
uname -a：查看系统内核版本及硬件架构

top：实时监控进程资源占用（按q退出）

ps：查看进程状态

ps aux：显示所有进程详细信息

df -h：查看磁盘空间使用情况（-h以人类可读格式显示）

free -h：查看内存使用情况

##     权限管理
chmod：修改文件权限

chmod +x script.sh：赋予文件执行权限

chmod 755 file：设置所有者读/写/执行，其他用户读/执行

chown：修改文件所有者

chown user:group file：将文件所有者改为user，所属组改为group

##   网络操作
ping：测试网络连通性

ping -c 4 baidu.com：发送4个数据包后停止

ifconfig/ip addr：查看网络接口信息

curl/wget：下载文件

curl -O [https://example.com/file.zip](https://example.com/file.zip)：保存文件到当前目录

##   压缩与解压
.tar：打包/解包

tar -cvf archive.tar dir：打包目录

tar -xvf archive.tar：解包

.tar.gz：压缩/解压

tar -czvf archive.tar.gz dir：打包并压缩

tar -xzvf archive.tar.gz：解压

##     搜索与查找
find：按条件查找文件

find /path -name "*.txt"：在/path下查找所有.txt文件

grep：在文件中搜索字符串

grep "error" logfile：在日志中查找包含error的行

grep -r "keyword" /path：递归搜索目录下所有文件

提示：所有指令后加--help可查看详细用法（如ls --help）。



# 四、http协议
---

### HTTP请求报文结构（CTF高频考点）
HTTP请求由**请求行 → 请求头 → 空行 → 请求体**四部分构成：

1. **请求行**：`[请求方法] [URL路径] [HTTP版本] CRLF`  
常见考点：修改请求方法绕过检测（例如用`HEAD`方法替代`GET`，或用`OPTIONS`探测服务器支持的方法）

| **常用方法** | **特点** | **CTF考点** |
| :--- | :--- | :--- |
| GET | 参数拼接在URL后，无请求体 | 长度限制导致参数截断、信息泄露 |
| POST | 参数放在请求体中 | 请求头缺少Content-Type/Content-Length导致解析异常 |
| HEAD | 仅返回响应头，无响应体 | 绕过前端长度/内容检测，泄露头中的flag |
| OPTIONS | 探测服务器支持的请求方法 | CORS跨域配置漏洞考点 |


2. **请求头**：以`键: 值`格式存储请求附加信息，CTF中最常修改的请求头：
    - `Host`：指定请求的目标主机（常见于Host头攻击、虚拟主机绕过）
    - `Referer`：标记请求来源（常用于绕过"必须从指定页面跳转"的验证）
    - `User-Agent`：标记客户端浏览器/系统信息（常用于伪造"仅允许指定浏览器访问"的绕过）
    - `Cookie`：存储客户端会话信息（最常见考点：修改Cookie中的`is_admin=0`为`1`实现身份绕过）
    - `X-Forwarded-For`：标记客户端真实IP（常用于绕过服务器IP限制，伪造本地IP`127.0.0.1`）
    - `Content-Type`：标记请求体数据类型（常考`application/x-www-form-urlencoded`、`multipart/form-data`、`application/json`格式，格式错误会导致参数无法解析）
3. **空行**：单独的`CRLF（\r\n）`，是请求头结束、请求体开始的标志，缺少会导致解析错误。
4. **请求体**：GET方法无请求体，POST方法存放请求参数，允许为空。

---

### HTTP响应报文结构
HTTP响应由**状态行 → 响应头 → 空行 → 响应体**四部分构成：

1. **状态行**：`[HTTP版本] [状态码] [状态描述] CRLF`，核心考点是常见状态码的含义：

| **状态码分类** | **含义** | **常见状态码** |
| :--- | :--- | :--- |
| 1xx | 信息提示 | - |
| 2xx | 请求成功 | `200 OK`<br/>（请求成功） |
| 3xx | 重定向 | `301`<br/>永久重定向、`302`<br/>临时重定向、`304`<br/>资源未修改（命中缓存） |
| 4xx | 客户端错误 | `400`<br/>请求格式错误、`401`<br/>未登录、`403`<br/>权限不足、`404`<br/>资源不存在、`405`<br/>请求方法不允许 |
| 5xx | 服务器错误 | `500`<br/>服务器内部错误 |


2. **响应头**：常见需要关注的字段：
    - `Set-Cookie`：给客户端设置Cookie，用于会话标识
    - `Location`：配合3xx状态码，指示客户端重定向的目标地址
    - `Content-Type`：标记响应体的数据类型，告诉浏览器如何解析内容
    - `Content-Length`：标记响应体的长度，长度不匹配会导致截断或解析异常

### curl用法 
　　curl是一个和服务器交互信息（发送和获取信息）的命令行工具。 

　　　　查看http响应头：curl -i [http://www.xxx.com](http://www.xxx.com)

　　　　查看交互过程： curl -v [http://www.xxx.com](http://www.xxx.com)

　　　　GET请求： curl -X GET [http://www.xxx.com](http://www.xxx.com) （当发起http请求时，curl会默认发起GET请求，也可以"-X GET"方式指定。）

　　　　POST请求： curl -X POST [http://www.xxx.com](http://www.xxx.com) -d “参数”（当使用POST请求方式，需要通过指定“-d”，向服务器传递数据。）

　　　　调试：curl -v [https://www.xxx.com](https://www.xxx.com)     （-v参数输出通信的整个过程，用于调试。



### Cookie欺骗、认证、伪造 


cookie是由后端服务器端创建保存在客服端一种数据库结构。浏览器请求和后端服务端响应都会携带创建的cookie来回传输。cookie在传输时存储在请求头和响应头中。 

cookie的作用： 保存用户与后端服务器的交互状态。
